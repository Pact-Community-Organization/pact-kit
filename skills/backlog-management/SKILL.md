# Backlog Management

## Work Item Types
- **Epic** — Large feature spanning multiple sprints
- **Story** — User-facing capability (INVEST criteria)
- **Task** — Technical work item (non-user-facing)
- **Bug** — Defect requiring fix
- **Spike** — Research/investigation (time-boxed)

## Backlog Organization
```
Epics → Stories → Tasks
  │
  └─ Milestone grouping (2-week sprints)
```

## Sprint Planning
1. Review backlog priority (MoSCoW)
2. Estimate effort (S/M/L/XL)
3. Capacity check (agent availability)
4. Select stories for sprint
5. Decompose stories into tasks
6. Assign to agents via Orchestrator

## Definition of Ready
- [ ] User story in INVEST format
- [ ] Acceptance criteria (Given/When/Then)
- [ ] Dependencies identified
- [ ] Gas budget estimated (if applicable)
- [ ] No unresolved questions
