# Prioritize Backlog

## Process
1. List all open stories/issues
2. Apply MoSCoW classification
3. Plot on value/effort matrix
4. Identify dependencies and constraints
5. Recommend sprint contents

## Output
```markdown
## Backlog Priority — {date}

### Must Have (This Sprint)
| Story | Effort | Value | Dependencies |
|-------|--------|-------|-------------|

### Should Have (Next Sprint)
| Story | Effort | Value | Dependencies |
|-------|--------|-------|-------------|

### Could Have (Backlog)
| Story | Effort | Value | Dependencies |
|-------|--------|-------|-------------|

### Won't (Out of Scope)
- {item}: {reason}

### Sprint Recommendation
Sprint capacity: {N stories}
Recommended: {list with justification}
```
